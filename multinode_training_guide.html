R1Ray 方案 — 安装环境 & 准备数据

在每台机器上分别执行（登录到每台机器）：

安装环境

bash setup_env.sh
# 安装 ms-swift, deepspeed, ray

准备数据

每台机器各自下载和过滤数据，确保数据在所有机器的同一路径下：

python download_data.py
python filter_data.py
# 完成后数据在 data_filtered/ 目录下
记住数据路径

记下数据的绝对路径（如 /root/sft_multinode_v1/data_filtered）， 后面需要填入 train_config.yaml 的 dataset 字段。 所有机器上的路径必须一致。

R2Ray 方案 — 启动 Ray 集群

1. 选一台机器作为 Head 节点

bash start_ray.sh head
# 输出示例:
#   [ OK ] Ray Head 已启动!
#   Dashboard: http://10.10.17.54:8265
#   在每台 worker 机器上执行:
#     bash start_ray.sh worker 10.10.17.54

2. 在每台 Worker 节点加入集群

将上一步输出的 HEAD_IP 填入：

# 在 worker 机器 1 上:
bash start_ray.sh worker 10.10.17.54

# 在 worker 机器 2 上:
bash start_ray.sh worker 10.10.17.54

# ... 其他 worker 同理

3. 验证集群

在 Head 节点执行：

ray status
# 应显示所有节点和 GPU 总数

也可以打开浏览器访问 Ray Dashboard：http://<HEAD_IP>:8265

NCCL 自动探测 & 环境变量注入

start_ray.sh 启动时会自动完成：

  • 探测有 IP 地址的网络接口（自动排除 reth 等无 IP 的底层接口）
  • 检测 InfiniBand / RoCE 设备
  • 将配置保存到 .nccl_env 文件

train_ray.sh 启动训练时会自动完成：

  • 读取 .nccl_env 中的 NCCL 配置
  • 通过 RAY_RUNTIME_ENV 将环境变量注入到每个 Ray actor 进程

如果训练时出现 NCCL 通信错误，可运行 bash check_network.sh 手动排查， 或编辑 .nccl_env 手动指定接口名后重新运行 train_ray.sh。

R3Ray 方案 — 启动训练

1. 编辑训练配置

修改 train_config.yaml 中的 dataset 路径：

# train_config.yaml
model: Qwen/Qwen3-4B-Thinking-2507
dataset: /root/sft_multinode_v1/data_filtered   # ← 改为实际路径

2. 启动训练

在 Head 节点执行：

bash train_ray.sh

脚本会自动完成以下操作：

  1. 读取 .nccl_env 中的 NCCL 网络配置
  2. 构建 RAY_RUNTIME_ENV，将 NCCL/GLOO 环境变量注入到每个 Ray actor
  3. 显示集群状态和 NCCL 配置信息
  4. 调用 swift sft 启动训练，Ray 自动将任务分发到所有 GPU
关于 device_groups 配置

train_config.yaml 中的 nproc_per_node: 8 表示每台机器使用 8 张 GPU。 如果某台机器的卡数不同，需要修改此值。Ray 会根据集群中实际的 GPU 数量自动确定节点数。

手动覆盖 NCCL 配置

如果自动探测的网络接口不正确，无需重启 Ray 集群。 直接编辑 .nccl_env 修改 NCCL_SOCKET_IFNAME 等变量， 然后重新运行 bash train_ray.sh 即可。

# .nccl_env 示例内容
export NCCL_SOCKET_IFNAME=eth0
export NCCL_IB_DISABLE=0
export NCCL_DEBUG=INFO
export GLOO_SOCKET_IFNAME=eth0

R4Ray 方案 — 停止 Ray

训练结束后，在每台机器上执行：

bash stop_ray.sh

1文件说明

nodes.conf
唯一需要手动编辑的文件。填入 6 个 IP、SSH 用户名、数据和模型路径。
configure.sh
自动探测网络接口、IB 状态、端口，配置免密 SSH，生成 env.sh。
env.sh
由 configure.sh 自动生成的完整配置。所有脚本从这里读取参数。
setup_env.sh
安装 ms-swift 和 DeepSpeed。每台机器执行一次。
sync_data.sh
从 node0 将数据、脚本、配置同步到所有 worker 节点。
launch_all.sh
一键启动所有节点训练。node0 前台运行，其他 SSH 后台启动。
train_multinode.sh
核心训练脚本。接受 NODE_RANK 参数，由 launch_all.sh 调用。
check_network.sh
网络诊断工具。手动排查时使用。configure.sh 已内置同等功能。

2第 0 步：编辑 nodes.conf

这是唯一需要你手动编辑的文件。你需要知道每台机器的 IP 地址。

如何查看机器 IP

SSH 到每台机器上，运行以下任一命令：

# 方法 1: 最简单
hostname -I
# 输出: 10.0.0.101 172.17.0.1    ← 第一个通常是主 IP

# 方法 2: 查看所有接口详情
ip -br addr
# 输出:
# lo       UNKNOWN  127.0.0.1/8
# eth0     UP       10.0.0.101/24    ← 以太网接口
# ib0      UP       192.168.100.1/24 ← InfiniBand 接口 (如果有)
# docker0  DOWN     172.17.0.1/16    ← 忽略
选哪个 IP？

选所有节点互通的那个网段的 IP。如果所有机器的 IP 都是 10.0.0.x 格式，就用这些。 忽略 127.0.0.1（本地回环）和 172.17.x.x（Docker 网桥）。

编辑 nodes.conf

# 把 xxx 替换为实际 IP, 有几台就写几行
NODES=(
    10.0.0.101    # node0 (主节点)
    10.0.0.102    # node1
    10.0.0.103    # node2
    ...            # 需要更多节点就继续加行
)

SSH_USER=root
DATA_PATH=/data/redsearcher/data_filtered
MODEL_PATH=Qwen/Qwen3-4B-Thinking-2507
NPROC_PER_NODE=8

SSH 到你选为 node0 的机器上，进入脚本目录：

bash configure.sh

脚本会自动完成以下操作：

检测 IB检查是否有 InfiniBand/RoCE 硬件否
检测端口确认 29500 端口可用否
生成 env.sh写入所有自动探测的参数否

关于免密 SSH

如果集群已经配置了免密 SSH，这一步完全无需交互。 如果没有，脚本会自动生成密钥并 ssh-copy-id， 每台不通的机器需要输入一次密码。这是唯一无法完全避免的交互。

4第 2 步：安装环境

在 node0 本机先安装：

bash setup_env.sh

这会安装：

  • ms-swift (最新版)
  • deepspeed (最新版)
前提条件

每台机器已预装 Python、PyTorch (CUDA)、flash-attn。 setup_env.sh 不会重复安装这些。

5第 3 步：同步数据

bash sync_data.sh

脚本会依次对 node1 ~ node(N-1)：

  1. rsync 训练脚本 + env.sh 到远程
  2. rsync 训练数据 (parquet 文件) 到远程
  3. SSH 远程执行 setup_env.sh 安装 ms-swift
耗时提示

数据同步时间取决于数据量和网络带宽。 ~9000 条 parquet 数据通常在几分钟内完成。如果模型是本地路径 (非 HuggingFace 名称)， 你需要手动在 sync_data.sh 中添加模型目录的同步。

6第 4 步：启动训练

bash launch_all.sh

脚本流程：

  1. SSH 启动 node(N-1) → ... → node1 (后台)
  2. 在 node0 前台启动训练 (直接看到输出)
  3. 所有节点通过 MASTER_ADDR:MASTER_PORT 建立通信
所有节点必须在短时间内启动

PyTorch Distributed 有超时机制 (默认 30 分钟)。 launch_all.sh 会在几秒内启动所有节点，不用担心超时。

8常见问题排查

问题：NCCL error "Bootstrap : no socket interface found"

这是最常见的 NCCL 错误，表示 NCCL 找不到可用的网络接口。

原因：NCCL_SOCKET_IFNAME 被设为了一个没有 IP 地址的接口 （如 reth0、veth 等底层/虚拟接口），或者环境变量没有传递到训练进程中。

解决方法：

  1. 运行 bash check_network.sh 查看推荐的接口名
  2. 编辑 .nccl_env（Ray 方案）或 env.sh（SSH 方案），将 NCCL_SOCKET_IFNAME 改为有 IP 的接口（如 eth0）
  3. 重新运行训练脚本（无需重启 Ray 集群）

Ray 方案额外说明： Ray actor 是独立进程，shell 中 export 的变量不会自动传递。 train_ray.sh 已内置 RAY_RUNTIME_ENV 注入机制， 会自动将 .nccl_env 中的变量注入到每个 Ray actor。 如果你手动运行 swift sft，需要自行设置 RAY_RUNTIME_ENV。

问题：训练启动后卡住不动

原因 1：NCCL 找不到正确的网络接口。
解决：参考上一条 "Bootstrap : no socket interface found" 的排查方法。

原因 2：防火墙阻止了 NCCL 端口。
解决：检查 MASTER_PORT (29500) 以及 NCCL 动态端口是否放通。 临时关闭防火墙测试：systemctl stop firewalld

原因 3：某个节点启动失败。
解决：检查对应节点的日志 logs/nodeX.log

问题：NCCL error / NCCL timeout (其他情况)

设置 export NCCL_DEBUG=INFO (已默认开启) 查看详细日志。
如果报 IB 相关错误但没有 IB 硬件，确认 NCCL_IB_DISABLE=1。
如果多个网络接口导致混乱，尝试设置 NCCL_SOCKET_IFNAME 为精确的接口名。

NCCL 2.27+ 兼容性问题： 如果报 ncclInvalidUsage，可在 .nccl_env 中追加以下变量后重试：

# 逐个添加尝试, 不需要全部开启
export NCCL_P2P_DISABLE=1
export NCCL_SHM_DISABLE=1
export TORCH_NCCL_USE_COMM_NONBLOCKING=0
问题：OOM (显存不足)

128K 序列长度 + 全参数微调对显存要求极高。确保：
1. DeepSpeed ZeRO-2 已启用 (--deepspeed zero2)
2. Gradient checkpointing 已启用 (--gradient_checkpointing true)
3. 精度为 bfloat16
如果仍然 OOM，考虑降低 --max_length

问题：数据不一致导致挂起

所有节点上的数据文件必须完全一致。重新运行 bash sync_data.sh 确保同步。
如果使用 HuggingFace 名称下载模型，设置 export HF_HOME=/some/shared/path 避免各节点独立下载。

9训练参数说明

参数值说明
--train_type fullfull全参数微调
--deepspeed zero2zero2梯度+优化器状态分片到所有 GPU，减少跨机通信
--max_length131072128K tokens, 模型原生支持 256K
--per_device_train_batch_size1受 128K 序列长度限制
--gradient_accumulation_steps4Global BS = 1 × 总GPU数 × 4
--learning_rate5e-5初始学习率
--lr_scheduler_typecosine_with_min_lr余弦衰减, 最低 1e-6
--warmup_ratio0.03前 3% steps 线性 warmup
--gradient_checkpointingtrue以计算换显存
--attn_implflash_attnFlash Attention 2, 128K 必需
--save_steps50每 50 步保存 checkpoint
--save_total_limit5最多保留 5 个 checkpoint
关于 Global Batch Size 的变化

原单机脚本: 1 × 8 GPU × 16 = 128
多机脚本: 1 × 总GPU数 × 4 (如 6 节点 × 8 卡 = 48 GPU → GBS=192)
如需保持更接近原来的值，可修改 train_multinode.sh 中的 --gradient_accumulation_steps (设为 3 则 BS=144)。
