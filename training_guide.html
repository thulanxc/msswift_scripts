1 安装环境

在 H200 服务器上执行以下命令（或保存为 setup_env.sh 后 bash setup_env.sh）：

# 创建 conda 环境
conda create -n redsearcher_sft python=3.11 -y
conda activate redsearcher_sft

# 安装 PyTorch（先用 nvidia-smi 确认 CUDA 版本，选对应的 whl）
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124

# 安装 ms-swift (当前稳定版 3.12.5) 和 DeepSpeed
pip install ms-swift -U
pip install deepspeed -U

# 安装 Flash Attention（128K 长序列训练必需）
pip install flash-attn --no-build-isolation

2 上传数据

将 data_filtered/ 文件夹上传至 H200 服务器的某个路径，例如：

/data/redsearcher/data_filtered/
├── train-00000-of-00005.parquet
├── train-00001-of-00005.parquet
├── train-00002-of-00005.parquet
├── train-00003-of-00005.parquet
└── train-00004-of-00005.parquet
数据格式已兼容，无需预处理

数据采用标准 messages 格式（与 REDSearcher 发布的 HuggingFace 数据集一致）。 ms-swift 的 AutoPreprocessor 会自动处理：

  • messages → 标准 [{role, content}] 对话
  • system_prompt → 自动映射为 system 消息
  • meta → 未使用列，自动忽略

3 启动训练

将以下内容保存为 train_sft.sh，修改其中两个路径后执行：

#!/bin/bash
# ============================================================
# Qwen3-4B-Thinking-2507 Full-Parameter SFT
# 8×H200  |  global batch=128  |  128K context  |  cosine LR
# ============================================================

# ---- 修改以下两个路径 ----
MODEL_PATH="Qwen/Qwen3-4B-Thinking-2507"        # 或已下载到本地的路径
DATA_PATH="/data/redsearcher/data_filtered"      # parquet 文件夹路径
OUTPUT_DIR="./output/qwen3-4b-thinking-sft"

NPROC_PER_NODE=8 \
CUDA_VISIBLE_DEVICES=0,1,2,3,4,5,6,7 \
swift sft \
    --model ${MODEL_PATH} \
    --dataset ${DATA_PATH} \
    --train_type full \
    --torch_dtype bfloat16 \
    --bf16 true \
    --max_length 131072 \
    --num_train_epochs 3 \
    --per_device_train_batch_size 1 \
    --per_device_eval_batch_size 1 \
    --gradient_accumulation_steps 16 \
    --learning_rate 5e-5 \
    --lr_scheduler_type cosine_with_min_lr \
    --lr_scheduler_kwargs '{"min_lr": 1e-6}' \
    --warmup_ratio 0.03 \
    --weight_decay 0.01 \
    --gradient_checkpointing true \
    --attn_impl flash_attn \
    --deepspeed zero3 \
    --save_strategy steps \
    --save_steps 50 \
    --save_total_limit 5 \
    --logging_steps 1 \
    --output_dir ${OUTPUT_DIR} \
    --dataloader_num_workers 8 \
    --dataset_num_proc 8 \
    --use_hf true

执行：

conda activate redsearcher_sft
bash train_sft.sh

参数详解

参数值说明
--train_type full full 全参数微调（非 LoRA）。4B 模型 + 8×H200 显存充裕。
--per_device_train_batch_size 1 每卡 batch size。128K 序列长度下必须为 1。
--gradient_accumulation_steps 16 梯度累积。全局 batch = 1 × 8 × 16 = 128。
--learning_rate 5e-5 初始学习率。
--lr_scheduler_type cosine_with_min_lr 余弦衰减（带最小学习率下界）。
--lr_scheduler_kwargs {"min_lr": 1e-6} 学习率衰减到 1e-6 后保持不变。
--warmup_ratio 0.03 前 3% 步数线性 warmup。约 6 步（3 epochs 共约 210 步）。
--max_length 131072 128K tokens。模型原生 max_position_embeddings=262144，无需 rope_scaling。
--num_train_epochs 3 训练轮数。8937 条数据 / 128 batch ≈ 70 steps/epoch，3 epochs ≈ 210 steps。
--gradient_checkpointing true 梯度检查点。大幅降低激活值显存占用，128K 训练必需。
--attn_impl flash_attn flash_attn 启用 Flash Attention 2。将 attention 显存从 O(n²) 降至 O(n)，128K 训练必需。
--deepspeed zero3 zero3 ZeRO Stage 3：模型参数 + 优化器状态 + 梯度全部分片到 8 张卡。
--torch_dtype bfloat16 bfloat16 模型权重精度。H200 对 bf16 有原生硬件加速。
--use_hf true true 从 HuggingFace 下载模型。国内服务器可去掉此项改用 ModelScope。
--split_dataset_ratio 0.0 (默认) 不划分验证集，全部数据用于训练。
--dataset_num_proc 8 数据预处理并行进程数。纯文本模型建议调高加速 tokenization。
--save_steps 50 每 50 步保存 checkpoint。3 epochs 总共约 210 步，会保存约 4 次。
--save_total_limit 5 最多保留 5 个 checkpoint，防止磁盘爆满。
关于 Thinking 模型的特殊处理

ms-swift 会自动识别 Qwen3-4B-Thinking 为 thinking 模型。使用默认 loss_scale='default' 时， 所有轮次的 assistant 回复（包括 <think> 内容）都会参与 loss 计算，历史思维链内容不会被移除。 这正是训练 deep search agent 所需要的行为。

Agent 场景 Loss 计算（tool_response 已正确排除）

本数据中 tool_response 内容位于 role: "user" 消息中。 ms-swift 的 loss_scale='default' 天然排除所有 user/system 角色的消息， 因此 tool_response 不会参与梯度计算，无需额外配置。

role内容参与 Loss？
user用户问题 / <tool_response>否
assistant<think> + <tool_call> + 正文是
systemsystem_prompt（deep search assistant）否

显存预估

组件单卡占用 (ZeRO-3)说明
模型权重 (bf16) ~1 GB 4B params × 2 bytes / 8 卡，ZeRO-3 分片到所有卡
优化器状态 (AdamW) ~4 GB fp32 master weights + 一/二阶矩，ZeRO-3 分片到 8 卡
梯度 ~1 GB ZeRO-3 分片到 8 卡
激活值 (128K seq) ~30-80 GB Flash Attention + gradient checkpointing 大幅减少，视实际序列长度波动
总计 ~36-86 GB H200 单卡 141 GB，留有充足余量

4 推理验证

训练完成后，使用最终 checkpoint 进行交互式推理：

CUDA_VISIBLE_DEVICES=0 \
swift infer \
    --model ./output/qwen3-4b-thinking-sft/checkpoint-210 \
    --infer_backend transformers \
    --stream true \
    --temperature 0 \
    --max_new_tokens 4096
全参数训练推理说明

全参数训练（非 LoRA）的 checkpoint 是完整模型权重， 推理时使用 --model（而非 --adapters）指向 checkpoint 目录。 ms-swift 会从 checkpoint 中的 args.json 自动读取训练时的参数配置。

常见问题与故障排除

OOM（显存不足）

脚本已使用 ZeRO-3（参数+梯度+优化器全分片），如仍 OOM 可考虑： 降低 --max_length（如 65536）；或降低 --per_device_train_batch_size。

国内下载慢 / 网络问题

方案 A：去掉 --use_hf true，默认走 ModelScope 下载（国内镜像更快）。

方案 B：提前下载模型到本地：

huggingface-cli download Qwen/Qwen3-4B-Thinking-2507 --local-dir /data/models/Qwen3-4B-Thinking-2507

然后把 MODEL_PATH 改为本地路径。

DDP 报错 "Expected to mark a variable ready only once"

添加参数 --gradient_checkpointing_kwargs '{"use_reentrant": false}'， 或者直接使用 DeepSpeed（脚本已使用，通常不会出现此问题）。

数据预处理卡住

128K 长序列 tokenization 较慢，8937 条数据预处理可能需要 10-30 分钟。 如需跳过重复预处理，可加 --load_from_cache_file true。

flash-attn 安装失败

确保安装了与 PyTorch 版本匹配的 CUDA toolkit。可尝试： pip install flash-attn --no-build-isolation， 或从 GitHub Releases 下载预编译 wheel。

可选优化

Packing（序列打包加速）

将多个短序列打包到一条 128K 序列中，提高 GPU 利用率。额外添加以下参数：

    --packing true

适合数据长度差异大的场景。开启后训练更快、显存占用更稳定，但预处理时间会增加。

断点续训

如果训练中断，可以从最近的 checkpoint 恢复：

    --resume_from_checkpoint ./output/qwen3-4b-thinking-sft/checkpoint-100

TensorBoard 监控

训练日志默认写入 output_dir/runs/，启动 TensorBoard 查看 loss 曲线：

tensorboard --logdir ./output/qwen3-4b-thinking-sft --port 6006
